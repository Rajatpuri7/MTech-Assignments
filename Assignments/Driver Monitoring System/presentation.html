How The System Works

The system watches your face through the webcam every frame (~30 times/sec) and calculates two numbers — EAR (eyes) and MAR (mouth) — to decide if you're drowsy or yawning.

📷 Webcam captures a frame
↓
🤖 MediaPipe finds 468 points on your face
↓
📐 Calculate EAR (Eye Aspect Ratio) & MAR (Mouth Aspect Ratio)
↓
EAR < 0.20 for 15 frames? MAR > 0.60? No face?
⚠ DROWSY ALERT + Beep
⚠ YAWNING ALERT
⚠ NO FACE

Step 5 — Eye Aspect Ratio (EAR)

def get_ear(landmarks, eye_points, w, h):
  pts = []
  for i in eye_points:
    x = int(landmarks[i].x * w)  # pixel X
    y = int(landmarks[i].y * h)  # pixel Y
    pts.append(np.array([x, y]))

  V1 = np.linalg.norm(pts[1] - pts[5])
  V2 = np.linalg.norm(pts[2] - pts[4])
  H  = np.linalg.norm(pts[0] - pts[3])

  ear = (V1 + V2) / (2.0 * H)
  return ear
EAR = (V1 + V2) / (2 × H)
V = vertical height  |  H = horizontal width
Eye Open
≈ 0.30
Eye Blinking
≈ 0.15
Threshold
0.20

def get_mar(landmarks, w, h):
  top    = point(13)  # top lip center
  bottom = point(14)  # bottom lip center
  left   = point(78)  # left mouth corner
  right  = point(308) # right mouth corner

  mar = norm(top-bottom) / norm(left-right)
  return mar
MAR = Height / Width
mouth opening height ÷ mouth corner-to-corner width
Closed mouth: MAR ≈ 0.1–0.3  |  Yawning: MAR > 0.60

Step 4 — Variables & Thresholds Explained

EAR_THRESHOLD  = 0.20   # if EAR drops below this → eyes are closed
MAR_THRESHOLD  = 0.60   # if MAR goes above this  → mouth is open
DROWSY_FRAMES  = 15     # must be closed for 15 frames straight
closed_counter = 0      # counter starts at 0, resets if eyes open
VariableTypeWhat It DoesWhy It Matters
EAR_THRESHOLD float Cutoff value for EAR Below 0.20 = eyes considered closed
MAR_THRESHOLD float Cutoff value for MAR Above 0.60 = mouth considered wide open
DROWSY_FRAMES int Frame count required Prevents false alarm from normal blinking
closed_counter int Counts consecutive closed frames Resets to 0 the moment eyes open. Like a stopwatch.
💡 WHY 15 frames? — At ~30 fps, 15 frames = 0.5 seconds. A blink takes ~0.3s. So if eyes are closed for 0.5s+ it's definitely not a blink.

Step 7 & 8 — Webcam + Main Loop

# Open the webcam
cam = cv2.VideoCapture(0)
# 0 = first connected camera

# Loop runs FOREVER until Q pressed
while cam.isOpened():

  success, frame = cam.read()
  # success = True/False
  # frame   = one image from camera

  frame = cv2.flip(frame, 1)
  # flip = mirror like a selfie

  h, w, _ = frame.shape
  # h = height, w = width in pixels

  rgb = cv2.cvtColor(frame, COLOR_BGR2RGB)
  # OpenCV uses BGR, MediaPipe needs RGB

  result = face_mesh.process(rgb)
  # AI scans the frame, finds face

🎥 VideoCapture(0)

0 = default webcam. Change to 1 for external camera. Returns a camera object we call cam.

↩ while cam.isOpened()

An infinite loop. Each iteration = one video frame. This is how all real-time video programs work — process one frame at a time, very fast.

🎨 BGR vs RGB

OpenCV stores colors as Blue-Green-Red. MediaPipe expects Red-Green-Blue. cvtColor swaps the channels so colors don't appear weird.

Step 9 — Drowsiness & Yawn Detection Logic

# --- DROWSINESS CHECK ---
if ear < EAR_THRESHOLD:  # eyes closed?
    closed_counter += 1   # count frames

    if closed_counter >= DROWSY_FRAMES:
        status = "⚠ DROWSY! WAKE UP!"
        color  = (0, 0, 255)    # RED
        winsound.Beep(1000, 200)
        # 1000Hz tone for 200 milliseconds
else:
    closed_counter = 0  # reset → eyes opened

# --- YAWN CHECK ---
if mar > MAR_THRESHOLD:  # mouth wide open?
    status = "⚠ YAWNING DETECTED!"
    color  = (0, 165, 255)  # ORANGE
The Counter Trick:
Think of closed_counter as a timer. Every frame with closed eyes adds 1. If eyes open again, it resets to 0. Only when it hits 15 does the alarm fire.
Frame 1: EAR=0.12 → counter=1
↓
Frame 2: EAR=0.10 → counter=2
↓ ... keeps going ...
Frame 15: EAR=0.09 → counter=15 → 🚨 ALARM!
↓ (if eyes open mid-way)
Frame 8: EAR=0.32 → counter RESETS to 0
winsound.Beep(1000, 200)
First number = frequency in Hz (pitch of sound)
Second number = duration in milliseconds (200ms = 0.2 seconds)

The 3 Alert States

✅

🔴
DROWSY — Red Border + Beep
color = (0, 0, 255) — Eyes closed 15+ frames
🟠
YAWNING — Orange Border
color = (0, 165, 255) — Mouth wide open
Colors are BGR! (0,0,255) looks red, not blue, because OpenCV uses Blue-Green-Red order.
